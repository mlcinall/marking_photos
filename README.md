# Real Estate Photo Labeling Tool (PySide6)

Быстрое локальное desktop-приложение для разметки объявлений недвижимости по 3 классам:

- `0` — не нравится
- `1` — нравится
- `2` — не определено

Главный фокус этой версии: **скоростной keyboard-first UX** без браузерных перезагрузок и без медленного веб-потока.

---

## Что это и зачем

Это инструмент для длительной ручной разметки, когда у вас есть ZIP с папками объявлений и фотографиями.

Особенности:
- полностью локально (без БД и облака);
- project-based хранение (`data/projects/<project_id>/...`);
- просмотр только фото с индексами `0,2,4,...` после сортировки по имени;
- быстрые горячие клавиши в desktop GUI;
- Undo, редактирование старых меток, прогресс и статистика;
- автосохранение CSV + состояния проекта.

---

## Формат входных данных

ZIP должен содержать папки объявлений:

```text
dataset.zip
├─ 12345/
│  ├─ 001.jpg
│  ├─ 002.jpg
│  └─ ...
├─ 67890/
│  └─ ...
```

Допускается также один промежуточный корневой каталог внутри ZIP.

Поддерживаемые форматы:
- `.jpg`, `.jpeg`, `.png`, `.webp`

---

## Формат выходных данных

Для каждого проекта:

```text
data/projects/<project_id>/
├─ extracted/
├─ previews/
├─ logs/
│  └─ skipped.log
├─ metadata.json
├─ session_state.json
└─ results.csv
```

`results.csv` колонки:
1. `listing_id`
2. `shown_photo_indices`
3. `label`

## Установка и запуск (Windows, подробно)

### 1) Установите зависимости системы

1. Python 3.10+ (https://www.python.org/downloads/windows/)
2. Git (https://git-scm.com/download/win)

Проверьте:

```bat
python --version
git --version
```

### 2) Клонируйте репозиторий

```bat
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
cd marking_photos
```

### 3) Создайте и активируйте virtualenv

```bat
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Если PowerShell блокирует скрипты:

```bat
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

### 4) Установите Python-пакеты

```bat
python -m pip install --upgrade pip
pip install -r requirements.txt
```

### 5) Запустите приложение

```bat
python app.py
```

Откроется desktop-окно приложения.

## Быстрый сценарий разметки

1. Нажмите **«Импортировать ZIP в новый проект»**.
2. Выберите архив.
3. Начинайте листать фото клавишами.
4. После просмотра всех фото объявления нажмите `0`, `1` или `2`.
5. Приложение сразу перейдёт к следующему объявлению.

---

## Горячие клавиши (основной режим)

- `←` / `A` — предыдущее фото
- `→` / `D` — следующее фото
- `0` / `1` / `2` — присвоить класс
- `U` / `Backspace` — Undo

Эти горячие клавиши обрабатываются нативно через Qt shortcuts (без URL/query hacks).

---

## UX и скорость

Важные решения для производительности:
- нативный desktop-event loop (PySide6), без полной перезагрузки страницы;
- **нет записи на диск** при обычном листании фото;
- используются preview-изображения (`previews/`) вместо оригиналов;
- fit-to-window viewer с сохранением пропорций;
- тяжёлая таблица вынесена в отдельную правую панель и обновляется по важным событиям.

---

## Что сохраняется на диск, а что нет

Сохраняется на диск:
- импорт/создание проекта;
- присвоение или изменение метки;
- undo;
- открытие объявления в режиме редактирования.

Не сохраняется на диск:
- переходы между фото;
- временный курсор и листание.

---

## Редактирование уже размеченных объявлений

- В правой панели есть список размеченных объявлений.
- Двойной клик или кнопка «Открыть для редактирования».
- Вверху экрана явно отображается режим: `редактирование`.

## Типовые проблемы

### `ModuleNotFoundError: No module named 'PySide6'`

Решение:
```bat
pip install -r requirements.txt
```

### ZIP не импортируется

Проверьте:
- архив действительно ZIP;
- внутри есть папки объявлений;
- нет битого архива.

### После импорта 0 валидных объявлений

Проверьте:
- папки не пустые;
- есть поддерживаемые расширения;
- файлы не битые;
- есть читаемые even-index фото.

Смотрите `data/projects/<project_id>/logs/skipped.log`.
