# Приложение для разметки фотографий объявлений недвижимости

Локальное приложение на **Streamlit** для последовательной разметки объявлений по классам:
- `0` — не нравится
- `1` — нравится
- `2` — не определено

## Возможности

- Загрузка датасета через ZIP-архив.
- Чтение структуры: `id_объявления/фото...`.
- Детерминированная сортировка фото по имени.
- Показ только каждого второго изображения (индексы `0, 2, 4, ...`).
- Защита от битых файлов изображений (пропуск без падения приложения).
- Пропуск объявлений без валидных показываемых фото с логированием.
- Навигация по фото внутри объявления.
- Выбор класса только после просмотра последнего фото.
- Автосохранение:
  - результатов в `data/results.csv`,
  - состояния сессии в `data/session_state.json`.
- Восстановление прогресса после перезапуска.
- Undo последнего решения.
- Редактирование класса у уже размеченного объявления.
- Статистика по классам и общий прогресс.
- Горячие клавиши:
  - `←/→` или `A/D` — листать фото,
  - `0/1/2` — присвоить класс,
  - `U` или `Backspace` — undo.

## Структура проекта

- `app.py` — Streamlit-интерфейс и логика взаимодействия.
- `utils.py` — обработка ZIP, валидация изображений, построение индекса объявлений.
- `storage.py` — загрузка/сохранение сессии и CSV с результатами.
- `requirements.txt` — зависимости.
- `data/` — локальное хранилище (создаётся автоматически).

## Установка и запуск

1. Убедитесь, что установлен Python 3.10+.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Запустите приложение:

```bash
streamlit run app.py
```

4. В браузере:
   - откройте блок **«Импорт данных»**,
   - загрузите ZIP-архив,
   - нажмите **«Импортировать архив»**.

## Формат входных данных

Ожидается ZIP следующего вида:

```text
archive.zip
└── 12345/
    ├── 1.jpg
    ├── 2.jpg
    └── ...
└── 67890/
    ├── 1.png
    └── ...
```

- Каждая подпапка — отдельное объявление.
- Имя подпапки = `listing_id`.
- Поддерживаемые форматы: `jpg`, `jpeg`, `png`, `webp`.

## Выходные файлы

- `data/results.csv`:
  - `listing_id`
  - `shown_photo_indices` (JSON-массив индексов показанных фото)
  - `label` (`0|1|2`)

- `data/session_state.json`:
  - текущее состояние сессии,
  - позиция в объявлениях,
  - история действий для undo,
  - разметка и курсоры просмотра.

- `data/logs/skipped.log`:
  - лог пропущенных объявлений и битых изображений.

## Примечания по надежности

- Прогресс сохраняется автоматически после каждого действия.
- Приложение работает полностью локально и не требует БД.
- При случайном закрытии можно продолжить с места остановки.
