# Local Real Estate Photo Labeling Tool (Streamlit)

Удобное локальное приложение для разметки фото объявлений недвижимости в 3 класса:

- **0** = не нравится
- **1** = нравится
- **2** = не определено

Приложение рассчитано на **длительную ручную разметку** с автосохранением, Undo, проектной структурой хранения и продолжением с того же места после перезапуска.

---

## 1) Для чего нужен проект

Этот инструмент нужен, когда у вас есть ZIP-архив с объявлениями (каждое объявление — отдельная папка с фотографиями), и вы хотите быстро и последовательно разметить объявления для ML-пайплайна.

Ключевые особенности:

- Работает **полностью локально**, без БД и облака.
- Показывает только каждую вторую фотографию (индексы `0,2,4,...` после сортировки по имени).
- Автосохраняет результаты и состояние на каждом шаге.
- Поддерживает **Undo** и **редактирование уже размеченных объявлений**.
- Хранит данные разметки в структуре `data/projects/<project_id>/...`.

---

## 2) Формат входных данных (ZIP)

Поддерживается такой формат:

```text
dataset.zip
├─ 10001/
│  ├─ 001.jpg
│  ├─ 002.jpg
│  └─ ...
├─ 10002/
│  ├─ a.png
│  └─ ...
└─ ...
```

или вариант с промежуточной корневой папкой:

```text
dataset.zip
└─ export_2026_01_15/
   ├─ 10001/
   │  ├─ 001.jpg
   │  └─ ...
   └─ 10002/
      └─ ...
```

Где:

- имя папки (`10001`) = `listing_id`;
- внутри папки — фото объявления.

Поддерживаемые расширения:

- `.jpg`, `.jpeg`, `.png`, `.webp`

---

## 3) Формат выходных данных

Для каждого проекта создаются файлы:

```text
data/projects/<project_id>/
├─ extracted/                # распакованный датасет
├─ logs/
│  └─ skipped.log            # лог пропусков и битых файлов
├─ metadata.json             # метаданные проекта/импорта
├─ session_state.json        # состояние интерфейса и прогресс
└─ results.csv               # итоговые метки
```

`results.csv` содержит колонки:

1. `listing_id`
2. `shown_photo_indices` (JSON-массив индексов реально показанных фото)
3. `label` (`0`, `1`, `2`)

## 4) Полный запуск с нуля через GitHub (Windows, приоритет)

## Шаг 1. Установите Python

1. Перейдите на https://www.python.org/downloads/
2. Установите Python 3.10+.
3. **Важно:** при установке отметьте галочку **Add Python to PATH**.

Проверка:

```bat
python --version
```

Если не сработало, попробуйте:

```bat
py --version
```

## Шаг 2. Клонируйте репозиторий

Откройте **PowerShell** или **CMD**:

```bat
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
cd marking_photos
```

## Шаг 3. Создайте виртуальное окружение

```bat
python -m venv .venv
```

Если используете `py` launcher:

```bat
py -m venv .venv
```

## Шаг 4. Активируйте виртуальное окружение

PowerShell:

```bat
.\.venv\Scripts\Activate.ps1
```

CMD:

```bat
.\.venv\Scripts\activate.bat
```

## Шаг 5. Обновите pip и установите зависимости

```bat
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## Шаг 6. Запустите приложение

```bat
streamlit run app.py
```

Обычно Streamlit выводит URL вида:

- `http://localhost:8501`

Откройте его в браузере.

## 6) Как пользоваться приложением (пошагово)

## 6.1 Создание/выбор проекта

Слева в sidebar:

1. Блок **«Проекты»**.
2. Можно открыть существующий проект из списка.
3. Можно создать новый проект:
   - введите имя (необязательно),
   - загрузите ZIP,
   - нажмите **«Импортировать в новый проект»**.

Важно: новый ZIP создает **новый проект**, старые данные не стираются.

## 6.2 Импорт и проверка архива

После импорта приложение покажет сводку:

- сколько папок найдено,
- сколько валидных объявлений,
- сколько пропущено,
- причины пропусков.

В `logs/skipped.log` пишутся детали:

- пустые папки,
- папки без поддерживаемых изображений,
- битые изображения,
- случаи без валидных even-index фото.

## 6.3 Разметка объявления

Для текущего объявления вы видите:

- `listing_id`,
- статус (размечено/не размечено),
- текущую метку,
- прогресс: `Фото X из Y`,
- индексы показанных фото,
- прогресс просмотра: `Просмотрено N/Y`.

Классификация разблокируется **только после полного просмотра всех показываемых фото**.

## 6.4 Переход к следующему объявлению

После выбора класса (`0`, `1`, `2`) приложение:

- сразу сохраняет CSV и состояние,
- автоматически переходит к следующему неразмеченному объявлению.

## 6.5 Undo

Кнопка **Undo** (и горячая клавиша) отменяет последнее действие:

- возвращает предыдущую метку,
- открывает соответствующее объявление,
- ставит режим редактирования.

## 6.6 Редактирование уже размеченных

В интерфейсе есть:

- таблица по объявлениям,
- selectbox «Открыть объявление для редактирования».

Можно открыть любое размеченное объявление и сменить класс.

## 6.7 Продолжение после перезапуска

После закрытия приложения достаточно снова запустить:

```bat
streamlit run app.py
```

Откройте нужный проект — прогресс восстановится из `session_state.json`.

---

## 7) Горячие клавиши

- `←` / `A` — предыдущее фото
- `→` / `D` — следующее фото
- `0` / `1` / `2` — поставить класс
- `U` или `Backspace` — Undo

## 9) Типовые проблемы и решения

## Python не найден

Симптом: `python is not recognized`.

Решение:

- переустановите Python и включите `Add Python to PATH`,
- либо используйте `py` вместо `python`.

## pip не найден

Запускайте через модуль:

```bat
python -m pip install -r requirements.txt
```

## streamlit не запускается

Проверьте, что активировано venv и зависимости установлены:

```bat
pip show streamlit
```

Если пакет не найден — переустановите зависимости.

## Порт 8501 занят

Запустите на другом порту:

```bat
streamlit run app.py --server.port 8502
```

## Не устанавливаются зависимости

- проверьте интернет/прокси,
- обновите pip,
- попробуйте:

```bat
python -m pip install --upgrade pip setuptools wheel
pip install -r requirements.txt
```

## ZIP не импортируется

Проверьте:

- файл действительно `.zip`,
- внутри есть папки объявлений,
- нет повреждения архива.

## После импорта «0 валидных объявлений»

Проверьте структуру и контент:

- папки объявлений не пустые,
- фото поддерживаемых форматов,
- файлы не битые,
- для even-индексов есть читаемые изображения.

Посмотрите `logs/skipped.log` в папке проекта.

## 10) Инженерные гарантии надежности

- Безопасная распаковка ZIP (защита от path traversal).
- Очистка `extracted` через полное пересоздание директории.
- Атомарная запись `session_state.json` и `results.csv` через temp + replace.
- Валидация структуры состояния с версией (`state_version`).
- При повреждении состояния создается backup (`*.broken.json` / `*.invalid.json`) и показывается предупреждение.

## 11) Ограничения и особенности

- Приложение рассчитано на локальный запуск одним пользователем.
- Горячие клавиши работают через JS в браузере (без доп.сервера).
- Если изображения очень большие, отрисовка может быть медленнее.
